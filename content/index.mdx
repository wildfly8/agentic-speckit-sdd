## About Spec-Driven Development

SDD puts specifications at the center of development:

1. **Constitution** — project principles
2. **Specify** — what to build (user stories, requirements)
3. **Plan** — how to build it (tech stack, structure)
4. **Tasks** — actionable implementation steps
5. **Implement** — execute the plan

See `specs/001-nextra-docs-site/` in this repository for the full SDD artifacts.
